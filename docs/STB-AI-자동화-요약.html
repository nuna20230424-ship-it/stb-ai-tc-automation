1. 기획 — 왜 / 무엇을 / 어떻게

왜 (Why)

  • STB 매뉴얼 QA 300~500 TC 24/7 운용 부담
  • 펌웨어 릴리스 주기 단축 → 회귀 압박
  • "500 TC = 500 스크립트"는 무너짐 — 카탈로그 필요

무엇을 (What)

  • 카탈로그 v2 (메타데이터+표준 액션 DSL)
  • 3-tier Judge (임베딩→룰→vision 회색 지대만)
  • 증거 자동 번들 + JIRA 자동 등록

어떻게 (How)

  • Claude Code: 코드/스키마/도구 작성
  • Gemini: 시나리오 초안 / 엣지케이스 생성
  • Notebook(현장 게이트웨이) + Mac mini(중앙 분석) 분리

데이터 흐름 (정상 시나리오 1건)

  1. precondition 적용 (라이브 TV / 넷플릭스 로그인 등)
  2. steps 순차 실행 (IR/voice/wait/capture/navigate)
  3. 마지막 capture → embedding-mcp describe + text 임베딩
  4. detection-mcp 3-tier judge → verdict/tier/confidence
  5. 이상/회색 지대 → evidence-bundler 디렉토리 + JIRA 자동 등록
  6. InfluxDB catalog_runs measurement → Grafana

3-tier Judge 흐름 (detection-mcp v2)

화면 캡처 │ ▼ embedding-mcp /vision/describe + /text [임베딩] → Qdrant /query → best_score │ ├── score ≥ 0.96 → verdict=normal, tier=embedding ✅ 즉시 종료 ├── score < 0.85 → verdict=anomaly, tier=embedding ⚠️ 즉시 종료 └── 회색 지대 (0.85 ≤ score < 0.96) │ ▼ expected_keywords ↔ description [룰 매칭] → 키워드 N개 이상 포함? ├── yes → verdict=normal, tier=rule └── no │ ▼ vision 재질의 ("yes/no?") [vision tier] → 최종 보루 ├── yes → verdict=normal, tier=vision └── no → verdict=anomaly, tier=vision

4. 진행방식 — Phase별 산출물

Sprint 0 · Phase 0 (Week 1~2) 완료

인프라 발주 + 8 MCP 서비스 smoke. 노트북·맥미니 분리 아키텍처 확정.

Sprint 1 (Week 3~6) 완료

채널 Zap · Voice · Bluetooth 3 시나리오 자동화 + Grafana 1차 대시보드(stb-channel-zap).

Sprint 2 / Phase 1 (Week 7~12) 완료

카탈로그 v2 (36 시나리오 마이그레이션) + 작성 파이프라인 3종(drafter / edge-gen / excel-importer) + 안전 머지 도구.

Phase 2 — Judge 파이프라인 (2026-05-26) 코드 8/8 완료

  • detection-mcp v2 3-tier judge + evidence-bundler/viewer
  • 카탈로그 v2.1 expected_keywords 필드 + 36 시나리오 부여
  • baseline_vector_id 자동 시드 (--missing-only) + baseline-mcp scenario delete
  • Grafana judge-pipeline 대시보드 (tier 분포 / 회색 지대 / 카테고리별 confidence)
  • 골든셋 라벨링·튜닝·회귀 도구 (tools/golden_set/)
  • vision provider 벤치(4종) + Tier 3 다변화 (VISION_PROVIDER env)
  • 자체 골든셋 100장 라벨링 ← 실 STB 캡처 의존 (도구 준비됨)
  • 임계 튜닝 (위 100장 확보 시 1줄 실행)
  • vision 벤치 실행 → VISION_PROVIDER 결정

Phase 3 / Sprint 3 (Week 13~24) 예정

Self-Healing TC, Test Impact Analysis 본격, 외부 디바이스 팜 SaaS 검토, 100% 시나리오 커버리지, 부서 확산.

5. 사용가이드 — 명령어 치트시트

① 시나리오 추가

## Excel 매뉴얼 TC → draft 변환
python -m tools.excel_importer \
    --input manual_tcs.xlsx --out drafts/imported.json

## 엣지케이스 자동 생성
python -m tools.edge_case_generator \
    --base drafts/imported.json --variants 3

## 카탈로그 안전 머지 (충돌·검증·백업)
python -m tools.catalog.merge \
    --drafts drafts/imported.json --dry-run

② 베이스라인 시드 (Reference STB)

## 신규 시나리오만 빠르게 시드
python -m tests.baselines.seed_catalog \
    --firmware v1.2.3 --missing-only

## 펌웨어 업그레이드 후 재시드
python -m tests.baselines.seed_catalog \
    --firmware v1.3.0 --replace

③ 골든셋 라벨링 (DUT)

## evidence 번들에서 (권장)
python -m tools.golden_set.label_cli \
    --from-evidence evidence/<dir> \
    --labeler me@kaongroup.com

## 비대화 배치
python -m tools.golden_set.label_cli \
    --scenario epg_open_7day --image ./shot.png \
    --firmware v1.2.3 --labeler me@x.com \
    --verdict normal --tier embedding --yes

④ 임계 튜닝 + Vision 벤치

## 임계 grid search (HARD_NORMAL × HARD_ANOMALY)
python -m tools.golden_set.tune_thresholds \
    --objective balanced

## vision provider 비교 벤치
python -m tools.vision_bench.cli \
    --objective accuracy-first \
    --save-report reports/bench.json

⑤ 회귀 실행

## 전체 카탈로그 회귀 + evidence 자동 번들
cd tests && pytest -m catalog

## 카테고리/우선순위 필터
pytest -m "epg and not slow"
pytest -m catalog -k "priority_P1"

## 골든셋 회귀 (임계 변경 안전망)
pytest -m golden_set

⑥ Evidence 조회 (실패 트리아지)

## 최근 실패 목록
python -m tools.evidence.viewer list --limit 20

## 특정 시나리오 상세
python -m tools.evidence.viewer show 2026-05-26_epg

## JIRA 첨부용 zip
python -m tools.evidence.viewer export <prefix>

## 오래된 evidence 정리
python -m tools.evidence.viewer prune --older-than 30d

⑧ Grafana 대시보드

  • stb-channel-zap — 채널 Zap latency / Detection score / JIRA incident 어노테이션
  • stb-voice-bluetooth — 음성·블루투스 시나리오 트렌드
  • stb-judge-pipeline (Phase 2) — tier 분포, 회색 지대 비율(target < 10%), 카테고리별 confidence, 카테고리×tier 매트릭스
