TC 실행 흐름 — 4단계

사내 Excel TC를 카탈로그에 합쳐 자동 회귀까지 한 흐름. 각 명령은 한 번만 실행하면 이후 야간 자동 회귀에 자동 합류됨.

1TC Excel 업로드

사내 TC 엑셀을 docs/specs/에 복사. Mac 터미널 또는 Finder 드래그.

cp ~/Downloads/사내TC.xlsx docs/specs/

2카탈로그로 변환

컬럼이 다르면 --column-id "시험번호" 같은 플래그로 즉시 매핑.

python -m tools.excel_importer.importer \
  --input docs/specs/사내TC.xlsx \
  --output drafts/imported.json \
  --batch-size 8

→ id / category / priority / expected / sla_ms 자동 매핑. 자유텍스트 Test Steps는 LLM이 5종 액션 배열로 구조화 (API 키 있을 때).

3카탈로그 머지 (백업·검증 자동)

ID 충돌 검사, schema 검증, 자동 백업까지 한 번에. 실패 시 자동 롤백.

python -m tools.catalog.merge --drafts drafts/imported.json --on-conflict abort --dry-run   # 미리보기
python -m tools.catalog.merge --drafts drafts/imported.json --on-conflict abort           # 실제 반영

4실행 — 3가지 모드 중 선택

① 즉시 시험 실행 (특정 TC만):

pytest -m catalog -k "사내TC_001 or 사내TC_002"

② 영향 분석 회귀 (변경된 컴포넌트만):

python -m tools.tc_selector select --changed-components voice-asr --emit-pytest-k k.txt
pytest -m catalog -k "$(cat k.txt)"

③ 야간 무인 회귀 (GitHub Actions 자동, 매일 22:00 KST):

git push   # main에 머지되면 자동 — e2e-nightly.yml 워크플로

💡 어디서 실행하나? (장비별)

로컬 즉시운영자 노트북 터미널 (게이트웨이 MCP 가동 후)
야간 무인GitHub Actions e2e-nightly.yml @ self-hosted runner (notebook)
특정 TC만pytest -k + tc_selector emit-pytest-k
변경 영향만tc_selector → 자동 분석 후 야간 회귀에 합류 (70~90% 시간 절약)

TC 목록에서 시나리오를 클릭하면 여기에 상세가 표시됩니다.
