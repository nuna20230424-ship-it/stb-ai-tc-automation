# 04. Fast-Track 압축 일정 (가속 로드맵)

> 2026-05-23 추가. 원래 24개월 로드맵을 **6개월(24주)**로 압축한 빠른 실행 계획. 4배 가속.

## 가속의 3가지 핵심 레버

1. **SaaS 선도입** — Suitest/Witbe 라이선스로 인프라 구축 6개월을 6~8주로 단축
2. **병렬 실행** — 안 1·2·4·5를 직렬이 아닌 **동시 착수**
3. **MVP 좁히기** — 5개 시나리오 → **3개 핵심 시나리오**로 시작 (채널 Zap, EPG 탐색, OTT 진입)

> ⚠️ **트레이드오프**: SaaS 라이선스 비용 증가, 일부 차별화 기능 후순위, 초기 도메인 RAG 정확도는 누적 데이터로 보완

## Sprint 0 — Week 1~2 (즉시 착수)
**목표**: 첫 자동화 시나리오가 SaaS에서 돌아가는 상태.

- [ ] **SaaS 벤더 선정 PoC**: Suitest vs Witbe — 1주 평가 후 결정
- [ ] **HDMI 캡처카드·IR Blaster 발주** (1~2 세트, 기성품)
- [ ] **3개 우선 시나리오 정의**: 채널 Zap 시간 측정 / EPG 7일 탐색 / OTT 앱 진입
- [ ] **GitHub 저장소·CI 스켈레톤** (이미 완료, 본 저장소)

## Sprint 1 — Week 3~6 (빠른 가치 증명)
**목표**: 리더십에 "AI 자동화 작동" 데모.

- [ ] SaaS로 3개 시나리오 자동화 가동
- [ ] **안 1**: 자체 캡처+IR 인프라 1~2대 디바이스 구축
- [ ] **안 2 파일럿**: Claude Code로 Jira 티켓→Gherkin→Pytest 자동 생성 (1~2 케이스)
- [ ] Gemini Vision으로 화면 검증 1차 통합
- [ ] 첫 데모 (4주차 종료 시점)

## Sprint 2 — Week 7~12 (확장)
**목표**: Agentic QA 파이프라인 본격 가동.

- [ ] **안 2 본격**: 요구사항 자동 입력→테스트 생성→실행→리포트 풀 파이프라인
- [ ] **안 5 착수**: Claude Code + Gemini 멀티에이전트 오케스트레이션 (MCP 서버 구축)
- [ ] **Self-Healing 1차**: 실패 케이스 LLM 분석→자동 수정 재실행
- [ ] **RAG 도메인 학습**: 과거 결함 데이터·테스트 케이스 벡터화

## Sprint 3 — Week 13~24 (스케일)
**목표**: 부서 전체로 확장, 100% 자동화 달성.

- [ ] 안 5 완성 — 멀티에이전트 + Self-Healing + RAG 통합
- [ ] **안 3은 자체 구축 보류** → LambdaTest/BrowserStack 등 외부 디바이스 팜 SaaS로 대체 검토 (ROI 평가 후 결정)
- [ ] 시나리오 100% 커버리지 달성 (3개 → 전체)
- [ ] 다른 부서/제품 라인 확산
