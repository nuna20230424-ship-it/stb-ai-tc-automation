# 02. 구축 방안 5가지 비교

> 2026-05-23 작성. STB AI 테스트케이스 자동화를 위한 5가지 구축안을 정리한다.

## 요약 비교표

| 안 | 컨셉 | 초기 비용 | 구축 기간 | 차별성 | 추천 적용 시점 |
|---|---|---|---|---|---|
| 1 | Vision AI + 신호 주입 블랙박스 | 중 | 1~3개월 | 보통 | **1단계 PoC** |
| 2 | Agentic QA 파이프라인 | 중 | 3~6개월 | 높음 | 2단계 본격 확장 |
| 3 | 클라우드 디바이스 팜 | **매우 높음** | 6~12개월 | 높음 | 3단계 스케일 |
| 4 | SaaS 통합 + AI 레이어 | 라이선스 | 6~8주 | 낮음 | **빠른 ROI** |
| 5 | Claude+Gemini 멀티에이전트 Self-Healing | 중~높음 | 12~24개월 | **매우 높음** | 장기 비전 |

---

## ✅ 안 1 — Vision AI + 신호 주입 블랙박스 프레임워크 (실용 베이스라인)
- **구성**: HDMI 캡처카드 + IR Blaster + Claude/Gemini Vision API + Pytest/Playwright 래퍼
- **벤치마크**: stb-tester (오픈소스) + Eggplant 패턴
- **AI 역할**: 화면 캡처를 멀티모달 LLM이 검증(예: "EPG에 KBS1이 표시되는가?"), 로그를 LLM이 요약
- **강점**: STB 본질에 가장 충실, 디바이스 종속성 낮음
- **약점**: 캡처/IR 인프라 초기 투자
- **추천 적용**: 1단계 PoC로 가장 적합

## ✅ 안 2 — Agentic QA 파이프라인 (요구사항 → 테스트 자동 생성)
- **구성**: Jira/Confluence(요구사항) → Claude Code 에이전트가 Gherkin 시나리오 생성 → Gemini가 검증 → 실행 코드 자동 작성 → CI 통합
- **벤치마크**: testQuality, Autify, Mabl의 Agentic QA 아키텍처
- **AI 역할**: 멀티에이전트 협업 — 기획(Claude), 검수(Gemini), 실행(Claude Code)
- **강점**: "100% 자동화"의 진정한 의미 — 테스트케이스 작성까지 자동
- **약점**: 도메인 학습(RAG) 구축 필요, 초기엔 환각 검증 부담
- **추천 적용**: 2단계 본격 확장의 핵심 축

## ⚠️ 안 3 — Netflix 스타일 클라우드 디바이스 팜
- **구성**: 사내 STB 랩 → 디바이스 큐/오케스트레이터 → 클라우드 콘솔에서 원격 실행
- **벤치마크**: Netflix NTS, Comcast OCATS (오픈소스 활용)
- **AI 역할**: 실행 결과 분석/우선순위 결정, 회귀 영향도 예측
- **강점**: 대규모 병렬 실행, 다부서 공유 가능
- **약점**: 인프라 비용·구축 기간 가장 큼 (6~12개월)
- **추천 적용**: 조직 규모/예산이 클 때 3단계 목표

## ⚙️ 안 4 — SaaS 통합 + AI 레이어 (빠른 ROI)
- **구성**: Suitest 또는 Witbe(2026 Agentic SDK) 구독 + 자체 AI 분석 레이어
- **벤치마크**: FX Digital, Sky의 외부 도구 협업 모델
- **AI 역할**: SaaS가 실행, Claude/Gemini는 결과 해석·리포트·테스트 추천
- **강점**: 6~8주 내 가시적 성과, 멀티 디바이스 즉시 지원
- **약점**: 라이선스 비용, 벤더 종속(lock-in)
- **추천 적용**: 리더십에 빠르게 성과를 보여줄 때 최선

## 🚀 안 5 — Claude Code + Gemini 멀티에이전트 Self-Healing 프레임워크 (가장 야심찬 안)
- **구성**:
  - Claude Code = 코드 생성·실행·MCP 서버 오케스트레이션
  - Gemini = 멀티모달 화면 검증·대용량 로그 분석 (장기 컨텍스트 강점)
  - **MCP 서버**: STB 제어(IR/HDMI/ADB), 캡처카드, JIRA, TestRail 연결
  - 자가치유 로직: 실패 시 LLM이 원인 분류 → 코드 자동 수정 → 재실행
- **벤치마크**: 2026 Agentic QA 아키텍처 + RAG 패턴
- **강점**: "AI 바이브코딩" 비전에 가장 부합, 차별화 가능
- **약점**: 거버넌스/감사 로깅 설계가 핵심 과제
- **추천 적용**: 장기 비전(2~3년) 목표 아키텍처
